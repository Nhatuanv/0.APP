Act as a Principal Software Engineer specializing in cross-platform native desktop apps, real-time 3D graphics, WebGL/Three.js, Rust networking, and stage LED mapping tools.

We are building a high-performance desktop application called:

Stage Previs & LED Mapping App

The app is for Windows and macOS. It allows stage designers, visual operators, and LED technicians to create a 3D previs scene, import 3D models, build LED screen layouts, map textures/video previews onto LED surfaces, import Resolume XML mapping data, save projects locally, and share a lightweight view-only preview over the local network.

Use this main tech stack:

Desktop shell:
- Tauri v2
- Rust backend

Frontend:
- React
- TypeScript
- Vite
- React Three Fiber
- @react-three/drei
- Three.js
- Zustand

Backend:
- Rust
- Tauri commands
- Axum HTTP server
- WebSocket server for local preview sync
- Native file read/write through Tauri/Rust

Do not use a cloud backend for the MVP.
Everything should work locally.

====================================================
LOCAL NETWORK PREVIEW BEHAVIOR
====================================================

Desktop app:
- User clicks "Start Preview Server".
- Rust backend detects local IP.
- Rust backend starts HTTP/WebSocket server.
- Frontend receives preview URL.
- UI displays:
  "Preview available at http://192.168.x.x:18080"
- Other devices on same network can open the URL.
- Preview clients connect to WebSocket.
- Desktop sends camera state and active LED texture mapping state.
- Preview clients update their view.

Real-time sync:
- In StageViewport, observe camera position, target, zoom, and rotation.
- Throttle sync to avoid sending too many messages.
- Send camera sync message through Tauri invoke to Rust backend.
- Rust backend broadcasts it to all WebSocket clients.
- Also sync selected LED texture mapping and screen config.
